Project Title: Healthcare Appointment System

Task: Assignment 4 -Functional Mobile App Prototype (MVP Development).

Technology Used: Flutter, Android Studio, Dart.

Pattern in Architecture: MVC-style layered architecture.

Project Overview:
It is a prototype of a functional mobile app that was created as a follow-up on Assignment 3. The application is grounded on a Healthcare Appointment System concept and uses the same target users and main workflow as the UI/UX prototype.

Core Workflow:
Login/Home/Doctor List/ Appointment Booking/ Confirmation.

Features Implemented:

1. Parody user login with input verification.
2. Physician search/filter.
3. Appointment booking form
5. Booking details confirmation screen.

Data Handling:
The application reads local mock data of doctors and renders dynamic content in the UI. Booking and search results are updated with interactivity.

Validation and Feedback:
The app confirms obligatory fields and email format. It offers error messages on the invalid input, loading messages in the process of the actions and confirmation of success after booking.

Code Organization and Architecture:
The project is structured into UI, Business Logic and Data layers so that the separation of concerns and reusable code components is achieved.

Testing:
The application was run on emulator/Android device to check on navigation, responsiveness, validation and overall workflow performance.
